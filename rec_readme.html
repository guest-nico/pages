ニコ生新配信録画ツール（仮）

新配信仕様に対応した映像とコメントのニコニコ生放送録画ツールです。

使い方

最初に「ツール」メニューの「オプション」の「アカウント設定」タブからログイン設定が必要です。
放送URL欄にURLを入力して「録画開始」を押すと録画を開始します。
lvから始まる放送IDを引数にして起動した場合は起動と同時に録画を開始します。

セグメントファイルの結合
ファイルやフォルダを選択して「ニコ生新配信録画ツール（仮.exe」にドロップすれば結合が始まります。

コメント付きでの再生・保存方法

デフォルトの設定の場合、動画の保存先に.xmlファイルとしてコメントファイルが保存されます。こちらを読み込むことでコメント付きで再生や保存ができるかと思います。コメント付きで再生するにはakpg様のcommeonやputin様のこめたんぷれいやのようなプレイヤーがあります。また、動画にコメントを描画した状態で保存するにはpsi様のさきゅばすやakpg様のcommeinのようなツールがあります。他にも便利なものがあるかもしれません。

注意

  • 一般会員アカウントで追い出しや満員により接続が切れてしまう放送は、録画でも途切れてしまいます。
  • 限定放送や有料放送などでは、ブラウザで視聴するときと同様に視聴する権限を持っている必要があります
  • 補完設定をしていても補完ができないときがあります。

機能説明

機能についての説明です。

録画エンジン

  • 標準のHLS録画エンジンを使う
    • 外部ツールを使わずHLSで録画します。補完機能が使えます。録画中に接続が途切れても一つの動画ファイルとコメントファイルとして 出力します。画質が選択ができます。
  • 外部のHLS録画エンジンを使う
    • FFmpegやStreamLinkなどの外部ツールを使ってHLSを録画できます。標準のHLS録画に比べて軽いです。 補完機能を使うことはできず、録画中に再接続が発生した場合は新しいファイルが作られます。 「コマンド」には外部ツールを呼び出す際のコマンドを指定します。画質が選択できます。
    {i} : プレイリストm3u8のURL
    {o} : 出力ファイルパス、「{o}.flv」などのように「.拡張子」を指定すると、 出力ファイル名の拡張子を指定できます。
    （例）　ffmpeg -i {i} -c copy {o}.mp4

画質

標準のHLS録画エンジン、または外部のHLS録画エンジンを選択しているときに使用されます。 サーバー接続時に取得可能な画質のうち、優先順位の最も高い画質で録画します。 2020年7月10日現在、「音声のみ」は遅延が1.0または3.0に設定されている場合のみ有効になるようです。

補完機能

HLSとRTMPを両方使用して録画することで、HLSの接続が途切れてセグメント抜けが発生した際に 抜けていた箇所をRTMPで取得できていればRTMP側のデータで補填を試みます。 従ってHLSとRTMPのどちらでも取得できていなかった部分は補完ができません。 追っかけ再生を使用することで補完を試みます。従ってオプションの補完設定にプレミアム会員のアカウントが登録されている場合のみ追っかけ再生機能が有効になり、補完機能も有効になります。 標準のHLS録画エンジンを選択しているときのみ使用できます。

視聴機能

録画中に「視聴」ボタンを押すことで放送を視聴できる機能です。
「オプション」の「視聴」タブで外部のプレイヤーを指定することもできます。「オプション」の「録画方法」タブで「標準のHLS録画エンジンを使う」もしくは「外部のHLS録画エンジンを使う」が設定されている場合はHLSのURL、「RTMP録画エンジンを使う」が設定されている場合はRTMPのURLが再生できるプレイヤーを設定します。
外部のコメントビューアを使用する場合、引数に放送URLを指定して起動できるツールを設定します。

コメントの置換

「オプション」の「コメント置換リスト」を設定することでコメントを保存する際に指定した文字を置換することができます。設定には正規表現が使用できます。「変換前」の列には置換する前の単語を入力し、「変換後」の列には置換した後の単語を入力します。
例
変換前　「わこつ」　変換後「枠取りお疲れ様です」
コメント内の「わこつ」という文字が「枠取りお疲れ様です」に変更されます。

変換前　「vpos="\d+"」 変換後　「vpos="0"」
コメントに設定されているvposを0に変換して保存する為、全てのコメントが0秒目に受信されたものとして保存されます。

変換前　「premium="24"」　変換後「premium="0"」
ブラウザ上で薄く表示されるコメントを通常コメントとして保存します。

変換前　「.+/gift.+」　変換後「」
コメント内の「/gift」が含まれる行を削除します。同様に「.+わこつ.+」「.+/emotion.+」などとすると、それぞれの含まれる行が削除されます。

変換前　「>/info \d+ (.+?)<」　変換後「>/nicoad {"message":"$1"}<」
変換前　「>/emotion (.+?)<」 変換後「>/nicoad {"message":"$1"}<」
変換前　「/gift .*?"(.+?)" (\d+)(.*?"){3}(.+?)".*<」　変換後「/nicoad {"message":"$1さんがギフト「$4($2pt)」を贈りました"}<」
/info、/emotion、/giftのコメントをニコニ広告のような形式で保存します。

置換は保存される文字に対して行われる為、json形式で保存する場合は、json形式に合わせて設定をする必要があります。

ログイン

次のブラウザとクッキーを共有する

ブラウザからクッキーを取得します。リストからブラウザ名を選択していただくと、そのブラウザのクッキーの取得を試みます。ブラウザ名の横に括弧書きでアカウント名が表示されている場合、そのブラウザからクッキーが取得されていることを表しています。
2024年11月現在、Google ChromeやMicrosoft Edge等、Chromium系のブラウザではクッキーを取得できなくなっています。
関連：ブラウザからクッキーを取得できない

ブラウザとクッキーを共有せず、次のアカウントでログインする

メールアドレスとパスワードからログインを試みます。二段階認証が設定されている場合、ログイン時は二段階認証の確認コードの入力が必要になります。また、ブラウザ上でログインしているアカウントと同じアカウントがこちらに設定されている場合、こちらの設定で何度かログインをしているとブラウザ側でログアウトが発生してしまうことがあります。

user_sessionクッキーを指定してログインする

ニコニコに接続する際に使用される、user_sessionクッキーを指定してログインします。ブラウザからログインしている状態で「https://www.nicovideo.jp/」を開いていただくと、ブラウザの開発者ツールからこちらのクッキーを確認できるかと思います。

関連：ログインができない、クッキーを取得できない

アップデートの方法

新しいバージョンの全てのファイルをこれまで使っていたフォルダに上書きコピーすると設定を引き継いでアップデートできます。

Tips

セグメント抜けが多い、他の作業をしていると途切れやすい

「ツール」→「設定」→「録画方法」タブの録画エンジンの設定で、「外部のHLS録画エンジンを使う」か「RTMP録画エンジンを使う」を指定すると改善できるかもしれません。また、標準のHLS録画エンジンを使う場合、録画する画質を下げたり補完機能を使わないように設定すると改善するかもしれません。

設定画面が開かない

動作には.NET Frameworkのバージョン4.5.2以上が推奨となっております。もしインストールされていない場合はインストールしてみると開けるかもしれないです。

タイムシフトのコメント取得数がブラウザの視聴ページ上の数より少ない

ページ上に表示されているコメント数はアリーナ席以外の部屋のコメントも含まれていますが、タイムシフトの録画時に取得しているコメントはアリーナ席のみとなっており、その内で追い出しコメントは保存をしないため、ページ上の数よりも保存される数は少なくなります。

コメントファイルを外部ツールで読み込めない

コメントファイルの末尾に「</packet>」を書き加えると読み込めるようになるかもしれません。

音声が再生されない

MPEG2Repairというツールを使用すると音声が再生できるようになるかもしれません。コミュニティ掲示板1604-も参照。
MPEG2Repair
http://www.digital-digest.com/software/MPEG2Repair.html

ファイル名が放送IDだけになる

ファイル名が長くなりすぎた場合、放送IDのみのファイル名で保存されることがあります。

mp4に変換すると音ずれしてしまう

タイムシフト・追っかけ再生で設定した時間と実際に取得したファイルに差がある

・5秒ごとの区切りについて
2023年4月20日現在、タイムシフト・追っかけ再生は約5秒ごとのファイルに分割されているようです。そのため、「タイムシフト録画設定」「追っかけ録画設定」画面で「録画終了時間」に0時間0分8秒のように設定している場合にも0時間0分10秒までのファイルとして取得されることがあると考えられます。区切りの時間位置はちょうど5秒毎の位置ではない場合もあるようでした。そのため、放送により同じ設定でも取得される時間位置が変わることもあるかもしれません。
また、過去には5秒以外の秒数で区切られている放送もあったようですので、現在もそのような放送があるかもしれません。

5秒ごとのファイルの開始位置が、ちょうどの時間は含まず、「録画開始時間」よりも後であり、「録画終了時間」よりも前にあるファイルが保存されます。「録画開始時間」に0時間0分5秒、「録画終了時間」に0時間0分13秒のように設定されている場合、0時間0分5秒から0時間0分13秒の間に始まった5秒間のファイルが保存されることになります。5秒ごとにファイルが区切られている場合は0時間0分5秒は含まれないため0時間0分10秒から0時間0分15秒までの5秒間のファイルが保存されることになるかと思います。

録画登録ツール（仮

リスト形式で放送を登録して新配信録画ツール（仮.exeを呼び出します。新配信録画ツール（仮と同じフォルダに配置します。

使い方

放送URL欄に放送URL等を入力して「録画リスト追加」ボタンを押すと録画リストに放送が追加されます。ユーザーページのURL、チャンネルID、コミュニティIDを入力するとタイムシフトが公開されているそれぞれの放送が追加されます。「録画開始」を押すと録画リスト内の放送の録画を開始します。

機能説明

機能についての説明です。

引数

録画登録ツール（仮.exe [設定]

「-設定項目=設定値」の形式で指定します。
起動オプションを指定できます。
例： -rec -addreserve

主な設定項目

-rec
起動時に「録画開始」ボタンの処理を行う
-addreserve
起動時に「タイムシフト予約から追加」ボタンの処理を行う
